# 🚀 План командной разработки IntiTrade (Backend + Frontend Roadmap)

Данный документ представляет собой пошаговый план совместной разработки проекта **IntiTrade** для команды из двух специалистов (Backend Developer и Frontend Developer). План структурирован по логическим этапом (фазам) без привязки к жестким временным неделям, что позволяет команде двигаться в оптимальном темпе.

---

## 👥 Распределение ролей и зон ответственности

| Специалист | Роль в проекте | Основные технологии |
| :--- | :--- | :--- |
| 🛠️ **Разработчик №1** | **Backend Developer** (Серверная часть, БД, безопасность, WebSockets) | Node.js, Express, TypeScript, Prisma ORM, PostgreSQL, Socket.io, JWT, PM2, Nginx. |
| 🎨 **Разработчик №2** | **Frontend Developer** (Пользовательский интерфейс, UX/UI, адаптивность) | React, TypeScript, Vite, Tailwind CSS, Lucide Icons, Axios, Socket.io-client. |

## 🗺️ Этапы разработки (Development Roadmap)

### 📌 Этап 1: Инициализация, Авторизация и База Данных (Auth & Core Foundation)
*Цель этапа: Построить фундамент приложения и обеспечить закрытый студенческий доступ.*

| Фаза этапа | 💬 Командное обсуждение (Group Discussion) | 🛠️ Задачи Backend Developer | 🎨 Задачи Frontend Developer |
| :--- | :--- | :--- | :--- |
| **Шаг 1.1**<br>Архитектура | Согласование архитектуры клиент-сервер, структуры базы данных (ER-диаграмма) и REST API контрактов для регистрации и входа. | Настройка локального сервера PostgreSQL. Инициализация проекта Express + TypeScript. Написание базовой схемы `schema.prisma` (`User`, `Token`). | Создание базового проекта React + Vite + TypeScript. Настройка Tailwind CSS и конфигурация цветовой палитры кампуса. Создание роутинга (`React Router`). |
| **Шаг 1.2**<br>Верификация | Утверждение политики безопасности: валидация регистрации строго по домену `@student.newinti.edu.my`. Обсуждение UX подтверждения почты. | Реализация эндпоинтов `POST /api/auth/register` и `/login`. Подключение сервиса отправки email-писем с кодом подтверждения. | Верстка страниц входа (`LoginPage`) и регистрации (`RegisterPage`). Настройка валидации полей ввода и отображения ошибок. |
| **Шаг 1.3**<br>Интеграция | Сквозное тестирование авторизации. Проверка работы приватных роутов и обработки истечения срока действия токенов. | Реализация JWT middleware для проверки токенов и защиты приватных эндпоинтов. Обработка CORS и безопасности заголовков. | Подключение Axios клиента с интерцепторами для JWT. Реализация глобального состояния (`AuthContext`). Экран успешной верификации почты. |

| Фаза этапа | 💬 Командное обсуждение (Group Discussion) | 🛠️ Задачи Backend Developer | 🎨 Задачи Frontend Developer |
| :--- | :--- | :--- | :--- |
| **Шаг 2.1**<br>Модели каталога | Согласование структуры объявлений (разделение на Товары и Услуги), списка категорий и серверной пагинации. | Расширение `schema.prisma` моделями `Listing` и `Category`. | **Шаг 2.2**<br>Поиск и фильтры | Проектирование фильтров: поиск по словам, фильтрация по категориям, состоянию (`NEW`, `GOOD`) и сортировка цен. | Добавление сложной фильтрации в Prisma-запрос получения объявлений. Добавление кнопок пагинации (← Prev / Next →) и скелетон-лоадеров (`Skeleton UI`). |

| Фаза этапа | 💬 Командное обсуждение (Group Discussion) | 🛠️ Задачи Backend Developer | 🎨 Задачи Frontend Developer |
| :--- | :--- | :--- | :--- |
| **Шаг 3.1**<br>Мастер создания | Обсуждение UX добавления товара: разбиение формы на 5 простых шагов для заполнения за < 2 минуты. | Реализация эндпоинта создания объявления `POST /api/listings`. Настройка безопасной загрузки и хранения фотографий. | Разработка пошагового мастера `CreateListingWizard` с индикатором прогресса и предварительным просмотром перед публикацией. |

## 🎯 Правила эффективной работы команды:
1. **Синхронизация по контрактам**: Перед началом каждого этапа Backend и Frontend разработчики сначала договариваются о точных JSON-контрактах запросов и ответов.
2. **Параллельная работа**: Пока Backend пишет логику и эндпоинты, Frontend верстает интерфейс с использованием моковых (фейковых) данных или Postman.
3. **Регулярные коммиты**: Каждая завершенная фича сразу пушится в общую ветку репозитория `GitHub`.
