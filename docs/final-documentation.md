**Университет**: INTI International University & Colleges

## 🌟 1. Видение и предназначение проекта (Product Vision)

**IntiTrade** — это закрытый инновационный университетский маркетплейс, созданный специально для студентов, преподавателей и сотрудников INTI International University.

### Проблематика (Problem Statement)
Студенты регулярно сталкиваются с необходимостью продажи или покупки подержанных учебников, электроники, бытовой техники и мебели (особенно при заселении или выселении из общежития). Использование открытых общедоступных платформ (таких как Facebook Marketplace, Avito или Carousell) сопряжено с рядом критических проблем:
* Высокий риск мошенничества и фейковых аккаунтов.
* Необходимость встреч с незнакомыми людьми за пределами безопасной территории кампуса.
* Отсутствие фильтрации по специфическим студенческим нуждам (например, поиск конкретных учебников по кодам дисциплин INTI или студенческих услуг).

### Решение (Proposed Solution)
IntiTrade формирует безопасную, доверительную p2p-экосистему строго внутри университетского сообщества:
1. **Эксклюзивный доступ**: Регистрация и вход возможны только при подтверждении владения активным корпоративным email-адресом университета (`@student.newinti.edu.my`).
2. **Безопасные зоны встреч (Safe Campus Meetup Zones)**: Платформа интегрирует рекомендации по проведению сделок исключительно в людных, охраняемых и хорошо освещенных точках кампуса (библиотека, кафетерий, студенческий центр).
3. **Обмен товарами и услугами**: Платформа поддерживает как физические товары, так и студенческие услуги (репетиторство, помощь с переездом, ремонт техники, дизайн).
4. **Нулевая комиссия (Zero Platform Fees)**: Все прямые сделки между студентами происходят без финансовых сборов со стороны системы.

## 👥 2. Роли пользователей и права доступа (User Roles & Permissions)

| Роль | Описание и полномочия |
| :--- | :--- |
| 👁️ **Гость (Guest)** | Неавторизованный пользователь. Имеет право просматривать главную ленту объявлений и детальные страницы товаров. Для отправки сообщений, добавления в избранное или публикации объявлений требуется вход. |
| 🎓 **Верифицированный студент (Student)** | Пользователь, подтвердивший университетский email. Может создавать объявления (5-шаговый мастер), вести чат с продавцами/покупателями в реальном времени, управлять статусом своих продаж, добавлять товары в избранное и отправлять жалобы на подозрительные публикации. |
| 🛡️ **Администратор (Admin)** | Сотрудник или модератор университета. Осуществляет проверку новых объявлений перед публикацией (одобрение/отклонение с указанием причины), обрабатывает жалобы пользователей, управляет категориями и имеет право блокировать аккаунты нарушителей. |

## 🏗️ 3. Архитектура системы (System Architecture)

Система спроектирована по современной клиент-серверной архитектуре с поддержкой двунаправленного обмена данными в реальном времени.

```
+-------------------+             REST API (JSON / HTTPS)             +--------------------+
|                   | <---------------------------------------------> |                    |
|   React Frontend  |                                                 |  Node.js + Express |
|  (Vite + Tailwind)| <---------------------------------------------> |    Backend API     |
|                   |           WebSockets (Socket.io Realtime)       |                    |
+-------------------+                                                 +--------------------+
                                                                                 |
                                                                                 | ORM (Prisma)
                                                                                 v
                                                                       +--------------------+
                                                                       |     PostgreSQL     |
                                                                       |  Relational DB     |
                                                                       +--------------------+
```

### Технологический стек:
* **Frontend**: React 18, TypeScript, Vite, Tailwind CSS, Lucide Icons, React Router DOM.
* **Backend**: Node.js, Express, TypeScript, Socket.io (для WebSockets), JSON Web Tokens (JWT).
* **База данных**: PostgreSQL, Prisma ORM.
* **Инфраструктура и Деплой**: PM2 (Process Manager), Nginx (Reverse Proxy), Linux Server (`adellina.adilkan.com`).

## 🗄️ 4. Структура Базы Данных (Database Schema)

В основе платформы лежат строго типизированные реляционные модели Prisma:

1. **`User` (Пользователь)**:
   * Идентификационные данные: `id`, `email`, `name`, `passwordHash`, `role` (`STUDENT` / `ADMIN`).
   * Профиль студента: `faculty` (Факультет), `campusArea` (Зона кампуса/Общежитие), `bio`, `avatarUrl`.
   * Статус: `isVerified` (Подтвержден ли email), `isBlocked`.

2. **`Listing` (Объявление)**:
   * Классификация: `title`, `description`, `price` (в валюте **RM**), `type` (`PRODUCT` / `SERVICE`), `categorySlug`.
   * Детали сделки: `condition` (`NEW`, `LIKE_NEW`, `GOOD`, `FAIR`), `isNegotiable` (Торг уместен), `location`, `meetupPreference`.
   * Модерация и метрики: `status` (`PENDING`, `ACTIVE`, `SOLD`, `REJECTED`, `ARCHIVED`), `rejectionReason`, `viewsCount`.

3. **`Conversation` & `Message` (Система обмена сообщениями)**:
   * Связывает `buyerId`, `sellerId` и конкретный `listingId`.
   * Обеспечивает изолированные чаты по каждому отдельному товару с фиксацией времени отправки.

4. **`AdminActionLog` & `Report` (Аудит и безопасность)**:
   * `Report`: фиксация жалоб от пользователей с указанием причины (`reason`).
   * `AdminActionLog`: полный аудит действий администратора (одобрение, отклонение, блокировка) для прозрачности модерации.

## 🎨 5. Спецификация пользовательского интерфейса (UX/UI Specification)

Дизайн платформы вдохновлен ведущими современными маркетплейсами (**Vinted, Depop, OfferUp**) с упором на визуальную привлекательность и удобство использования на смартфонах.

### Ключевые страницы интерфейса:
1. **Главная лента (`MarketplacePage`)**:
   * Интерактивные чипсы категорий для фильтрации в один клик.
   * Поисковая строка и панель расширенных фильтров (по типу, состоянию и сортировке цены).
   * Карточки товаров с четким отображением цены в **RM**, бейджем "Negotiable" и отметкой верификации продавца.
2. **Мастер создания объявления (`CreateListingWizard`)**:
   * Пошаговый 5-экранный процесс, позволяющий опубликовать объявление менее чем за 2 минуты.
   * Интуитивный выбор состояния товара карточками и указание безопасных зон встречи на кампусе.
3. **Детальная страница товара (`ListingDetailPage`)**:
   * Фотогалерея с переключением миниатюр и счетчиком просмотров.
   * Специальный визуальный блок **"Campus Meetup Zone"** с предупреждением о безопасности правил торговли.
4. **Личный кабинет продавца (`MyListingsPage`)**:
   * Управление жизненным циклом товара в один клик (отметка "Продано", перенос в архив).
   * Вывод причины отклонения модератором в виде яркого предупреждения для быстрой исправления ошибок.
5. **Мессенджер (`MessagesPage`)**:
   * Закрепленная в шапке карточка товара с ценой и статусом, предотвращающая путаницу в диалогах.
   * Мгновенная доставка сообщений в виде современных чат-бабблов.
